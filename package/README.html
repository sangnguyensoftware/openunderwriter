OpenUnderwriter @major.version@.@minor.version@ @increment.version@ - Installation Notes

Welcome to OpenUnderwriter and thank you for your interest in this project.

The notes included here are intended to get you up and running with OpenUnderwriter in the shortest possible time; around 10 minutes if you already have the prerequisites installed.

Beyond this basic configuration there are a host of other more detailed customizations that are possible; some of which should be considered necessary before you think about using OpenUnderwriter in a production environment. For now however, the instruction here should be enough to get you off the ground.

Note: You will need to be happy working from a command line in order to complete the installation. If you're not, please consider taking a look at our online demonstration, or book a personal demonstration instead.

Prerequisites

  • Java 8 Development Kit (aka Java SE) available from Oracle or other sources depending on your platform.
  • MySql Database server.

Installing the Java Development Kit

  • Browse to Java 8 downloads.
  • Select the appropriate 64bit Java SE download for your environment, making sure it is Java SE 8. The OpenUnderwriter server can only be installed in 64bit environments.
  • Once downloaded, follow the instructions in the installation wizard.

Installing MySql

  • Browse to http://dev.mysql.com/downloads.
  • Select the appropriate download for your environment. You will need version 5.7.
  • Once downloaded, follow the installation wizard - as part of this you will be prompted to select a password. You will need this later on.

If you are installing on a Linux based machine, edit /etc/my.cnf and add the following line in the [mysqld] section: 'lower_case_table_names = 1'

Regardless of your platform, edit my.cnf and review the 'sql_mode' setting. It should be set as follows: `sql_mode=STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO, NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION`

If you see the error "MySQL cannot be executed. Please make sure that the [MySQL HOME]/bin folder is on the PATH." during installation, these notes for Windows or MacOSX environments might help.

Installing OpenUnderwriter

OpenUnderwriter community is supplied as a zip file. Simply unpack this zip file and follow the instructions below (described under Starting the OpenUnderwriter server).

If you are installing on Windows, we recommend that you don't use Window's built-in zip file viewer. A number of issues have been reported on the forums which seem to point at file corruption caused by the build-in tool. We recommend that you use another tool like winzip, unrar, or jar to unpack the .zip file.

Starting the OpenUnderwriter server

Command scripts are supplied which should be suitable for MS Windows, and Unix users. From the command prompt, cd into openunderwriter-community-@major.version@.@minor.version@@increment.version@/bin, then either execute run.sh on Unix, or run.bat on MS Windows.

The first time you run OpenUnderwriter it will automatically setup your environment for you. Most importantly, it will create and populate databases in MySQL. During the setup process, you will be prompted for the database username and password. Use either the root user and the password created during MySQL installation, or the credentials of another use who has permission to create databases.

Depending on the spec. of your machine the server may take one or two minutes to start up. It will output progress messages as it starts. Start up is complete when a browser window opens showing the OpenUnderwriter home page, you will also see the following message in the console output:

Accessing OpenUnderwriter

Once the server is running, you can access OpenUnderwriter by pointing your browser at http://localhost:8080

Stopping the OpenUnderwriter server

Either closing the command prompt window, or hitting CTRL-C will close the server down cleanly.

Trouble shooting

If you have any issues during installation, or while you are using OpenUnderwriter, the best place to start looking for information is in our project forum project forum, and feel free to post a question if you can't find the answer you need.

If you believe you have found a bug please check our bug tracker. However, do please search carefully first, and only look at raising a new bug if you are sure your particular problem is not already covered.

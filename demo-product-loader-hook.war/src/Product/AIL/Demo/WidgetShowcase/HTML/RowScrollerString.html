The string type supports the entry of a single line of unformatted text (see note for multiple line formatted text support). The format options supported by the string attribute affect the rendering/validate in the follow ways:

PageFlow 
	<rowScroller [row scroller options]>
		<attributeField title="title" subTitle="sub title" binding="xpath binding" [condition="XPath expression"] [columnCondition="XPath expression"]/>
 	</rowScroller>
  • title Title rendered at the top of the column.
  • subTitle Rendered below the main title.
  • binding xpath binding referring to an Attribute in the model to which this column is bound.
  • condition An optional XPath condition that controls whether a given cell appears in the table or is hidden.
  • columnCondition An optional XPath expression that can be used to show/hide the entire column.
Model 
	<attribute id="id" value="value" format="string,[min=min length],[max=max length],[size=field size],[pattern=reg exp]/>
  • min Used during validation to define the minimum length of string that will be accepted.
  • max Used during validation to define the maximum length of string that will be accepted.
  • size Used during rendering to limit the size of the field that will appear on the user interface
  • pattern Used during validation, this option defines a regular expression that the content of the string will be validated against.

This widget gives the user read/write access to a party's contact systems - addresses, phone numbers, and email addresses. The PageFlow author has control over which party's contact systems are displayed and which types of contact systems should be included.

PageFlow 
<contactSystemDetails [binding="xpath"]
                      [classes="regular expression"] 
                      [types="regular expression"]
                      [renderHint="render hint"] 
                      [title="title"] />
  • binding - XPath expression selecting the party for which contact systems should be displayed.
  • classes - A regular expression used to filter the classes of contact system that will be included.
  • types - A regular expression used to filter the. contact system types that should be included.
  • render hint - Hint to be associated with the rendered widget.
  • title - A title to be applied to the rendered widget.
Examples

Display all contact systems associated with the broker.

<contactSystemDetails binding="/broker" />

Display only the postal addresses associated with the client.

<contactSystemDetails binding="/client" classes=".*\.Address" />

Display only the main postal addresses and phone numbers associated with the client.

<contactSystemDetails binding="/client" classes=".*\.(Address|PhoneNumber)" types="i18n_contact_system_main.*" />

On this page, the widget is configured to display the client's main address (postal) and phone number.

The BuyNowWithPayPalButtonAction provides the facility for PageFlows to authorise and collect premium payments using PayPal.

The button can be added to a navigation section as follows

     <navigationSection>
         <buyNowWithPayPalButtonAction destinationOnSuccessPageId="page id" destinationOnFailurePageId="page id"/>
         ...
     </navigationSection>

This widget represents itself on the user interface as the familiar "Buy now with PayPal" button. When the user clicks the button OpenUnderwriter will create a PayPal payment request and then forward the user the PayPal login screen to authorise the payment. From here the user can either cancel the request or authorise it. If the request is cancelled, the user is returned to OpenUnderwriter at a point in the PageFlow specified by the destinationOnFailurePageId attribute.

If the user authorises the payment, the payment is executed and the user is returned to OpenUnderwriter at a point in the PageFlow specified by the destinationOnSuccessPageId attribute. Executed payments are complete at will appear in the both the user's and merchant's transaction histories on PayPal.

Defining the amount to be paid

The policy's paymentDetail object defines the amount to be paid. The payment services inspect the collection of MoneyProvisions held in policy.paymentDetails and create requests for each instance that they find which has a paymentMethod of PayPal and a status of NEW. Based on the following example, one payment request would be made for £276.58.

     <policy>
         ...
         <paymentDetails>
             <description>Insurance premium payment</description>
             <moneyProvision>
                 <description>Insurance premium payment</description>
                 <frequency>ONE_TIME</frequency>
                 <number>1</number>
                 <paymentsStartDate>2014-05-21T16:23:28.705Z</paymentsStartDate>
                 <paymentsEndDate>2014-05-21T16:23:28.705Z</paymentsEndDate>
                 <amount amount="276.58" currency="GBP"/>
                 <paymentMethod xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:type="payPal"/>
                 <status xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:type="java:java.lang.String">NEW</status>
             </moneyProvision>
         </paymentDetails>
         ...
    </policy>

In a typical PageFlow the paymentDetails object is created and populated using a JavaService once the quotation calculations have been completed.

Once a request has been sent to PayPal for a MoneyProvision, the instance's status value is updated to REQUESTED.

Configuring a product to accept payments

In order for a product to accept PayPal payments it must first be configured with parameters defining the account into which payments will be made. These parameters are defined in the product's Registry.xml. The values may be defined in the root product of a hierarchy or set individually in the products themselves. This allows for each product to be configured with independent settings if necessary. The parameters involved are as follows:

<configuration>
    ...
    <group name="PaymentMethods">
        <group name="PayPal">
            <parameter name="Mode">mode</parameter>
            <parameter name="ClientID">Merchant's PayPal application client ID</parameter>
            <parameter name="ClientSecret">Merchant's PayPal application secret</parameter>
        </group>
    </group>
    ...
</configuration>
  • mode - Use "sandbox" while you are testing your product using mock-money, and "live" when you want to accept real payments.
  • Merchant's PayPal application client ID - The client ID generated by PayPal for the Merchant's application.
  • Merchant's PayPal application secret - The secret generated by PayPal for the Merchant's application.

PayPal sandbox login

By default the page is setup to use OpenUnderwriter's PayPal sandbox. You can login to the sandbox using the user: "client@openunderwriter.org" and the password "clientclient". Being a sanbox account, no actual financial transactions take place.

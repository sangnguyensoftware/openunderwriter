Document buttons are a set of specialised UI buttons which generate and display common documents.

The buttons will enable and disable automatically depending on the state of the Policy object being used by the PageFlow. When selected, they will either generate and then display the associated document or, if the document is already present, they will simply display it.

PageFlow

There are 6 buttons, one for each document type: 

<viewQuotationButtonAction />
<viewCertificateButtonAction />
<viewInvoiceButtonAction />
<viewWordingButtonAction />
<viewScheduleButtonAction />
<viewPolicyButtonAction />

Each button supports the same options: 

<viewQuotationButtonAction label="label" [renderHint="inline"] />
  • label - Override the default button label label.
  • render hint - By default the file will be downloaded with a Content-Disposition of attachment. Most browsers will respond by downloading the content and storing it in the user's Downloads folder. However, If a value of "inline" is used, the Content-Disposition will be set to "inline". Typically, browsers will now open the document in a new browser tab if they have a viewer for the file's mime type.
